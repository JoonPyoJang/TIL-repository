1. 버전정리를 한다는건 어떤 의미인가?
- 버전 관리란? 파일을 수정하고 업데이트를 하면서 변경된 항목에 대해 기록 하는것

2. 작업내역 단위인 commit에는 어떤 정보가 포함되어 있어야 잘 버전관리가 될까?
- 직접적인 수정 내용이 있어야 버전이 잘 관리가 된다.

- 버전관리란 프로그램 상태정보를 알고 있다는것 commit을 사용해 버전 관리를 한다.
- GIT 초기화는 프로젝트를 git에서 관리해주는 것을 git 초기화라고 한다.
- commit은 3가지 내용을 필수적으로 포함
1. 누가	2. 언제	3. 어떤작업을 했는지

- repository란 원격에 프로젝트를 저장해 프로젝트를 관리
- 클라우드에 따로 저장하여 로컬repo와  원결repo를 연결하는것을 트래킹이라고 한다.

-PUSH 로커렝서 원격으로 올려 주는것
-PULL 원격에서 로컬에 내려 받는것
